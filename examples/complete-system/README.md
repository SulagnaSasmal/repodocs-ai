# Complete System Example

## Scenario

This example shows how RepoDocs AI can document a payments capability end to end across product, feature, API, governance, and diagram assets.

## Workflow Demonstrated

1. Define the product and feature context.
2. Generate or draft API documentation from an OpenAPI source.
3. Review the output using governance and validation assets.
4. Publish the approved documentation to a docs site.

## Expected Outcome

Teams can use this folder as a reference implementation for how RepoDocs AI assets fit together in a realistic delivery flow.
