Control Plane

Repository automation now has a small hosted runtime surface.

RepoDocs AI can now expose validation, export, agent, analytics, and graph jobs through a small Node control plane with HTTP endpoints and a lightweight dashboard.

Endpoints

Runnable over HTTP

  • GET /health
  • GET /auth/status
  • GET /jobs
  • GET /runs
  • GET /artifacts
  • POST /jobs/validate
  • POST /jobs/export
  • POST /jobs/automation

Hosted Scope

Authenticated and queue-aware

The control plane is meant to bridge repo-native automation and a hosted deployment model without introducing a full platform stack. It now protects job and artifact endpoints with API keys, accepts multiple requests through a FIFO queue, keeps artifacts on disk, and can be packaged into a container image.

Runtime

Start it locally or run it in a container.

Run npm run control-plane:start with REPODOCS_CONTROL_PLANE_API_KEYS set, or build the container image with npm run docker:control-plane:build.
