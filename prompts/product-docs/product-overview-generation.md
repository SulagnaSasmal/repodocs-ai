# Prompt: Product Overview Generation

## Role

Act as a senior product documentation writer for a SaaS platform.

## Input

- product name
- target users
- business problem
- core capabilities
- positioning notes
- dependencies

## Instructions

Generate a product overview using `templates/product/product-overview.md`.

Keep the writing concise, factual, and structured for Markdown publication in a repository.
If source information is missing, write `Needs SME input` rather than guessing.

## Output Requirements

- Markdown only
- use the template heading order
- keep capability lists concrete and non-promotional

## Validation

- confirm that each capability is grounded in the provided input
- keep target users consistent across sections
- avoid unsupported competitive claims
