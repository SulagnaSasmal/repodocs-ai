# Design-Partner Intake

Use this as the source for a Tally, Google Form, or Notion form.

## Goal

Qualify teams that may want an early design-partner conversation around API documentation generation and review.

## Questions

1. Name
2. Work email
3. Company
4. Role
5. What kind of API docs do you own today?
6. Do you already have an OpenAPI spec?
7. Where do your docs live today?
8. What's the biggest documentation bottleneck for your team right now?
9. Would you want help with:
   - generating docs from a spec
   - reviewing existing docs against a spec
   - setting up a docs-as-code repo
   - building an internal workflow for AI-assisted docs
10. How urgent is this problem?
11. Would you be open to a 30-minute call?
12. What kind of next step would be realistic for you if RepoDocs AI looks useful?

## Short Intro Copy

RepoDocs AI helps API teams generate, review, and validate documentation from repo-native assets and OpenAPI specs. If you want help applying it to a real documentation workflow, leave your details here for a short design-partner follow-up.

## Recommended Field Types

1. Name: short text
2. Work email: email
3. Company: short text
4. Role: short text
5. What kind of API docs do you own today?: long text
6. Do you already have an OpenAPI spec?: multiple choice with `Yes`, `Partially`, `No`
7. Where do your docs live today?: multiple choice with `GitHub`, `GitLab`, `Confluence`, `Notion`, `ReadMe`, `Stoplight`, `Other`
8. What's the biggest documentation bottleneck for your team right now?: long text
9. Would you want help with: multi-select checkboxes
10. How urgent is this problem?: multiple choice with `Urgent this quarter`, `Important this half`, `Exploratory`
11. Would you be open to a 30-minute call?: yes/no
12. What kind of next step would be realistic for you if RepoDocs AI looks useful?: multiple choice with `Just exploring`, `Feedback only`, `Future pilot when budget exists`, `Implementation help later`

## Tally Setup

1. Create a new form in Tally.
2. Set the form title to `RepoDocs AI Design-Partner Intake`.
3. Paste the Short Intro Copy into the form description.
4. Add the 12 questions above using the Recommended Field Types.
5. Mark questions 2, 8, 10, 11, and 12 as required.
6. Add a hidden field called `source` and set a default value such as `github-discussion`, `wtd-slack`, or `linkedin` when you share the form.
7. Set the thank-you page text to: `Thanks. I will review this and follow up if RepoDocs AI looks like a fit for a short design-partner conversation.`
8. Turn on email notifications for new responses.
9. Publish and copy the public link.

## Google Forms Setup

1. Create a blank Google Form.
2. Set the title to `RepoDocs AI Design-Partner Intake`.
3. Paste the Short Intro Copy into the description.
4. Add the same 12 questions using the Recommended Field Types.
5. Mark questions 2, 8, 10, 11, and 12 as required.
6. In Settings, collect email addresses only if you want Google to enforce an email field; otherwise keep the explicit Work email question.
7. Set the confirmation message to: `Thanks. I will review this and follow up if RepoDocs AI looks like a fit for a short design-partner conversation.`
8. Shorten the form address before sharing it publicly.

## Notion Setup

1. Create a new page called `RepoDocs AI Design-Partner Intake`.
2. Paste the Short Intro Copy at the top.
3. If your Notion workspace supports Forms, create a form database and map each question to the matching property type.
4. If Notion Forms isn't available in your workspace, publish the page with a `Contact me with these answers` template or use Tally instead.
5. Make sure the public share setting is enabled before you distribute the link.

## Recommended Default

Use Tally first. It's the fastest option for a clean public intake form, supports hidden source attribution, and requires the least manual setup.
