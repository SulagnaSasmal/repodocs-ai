# System Architecture: {system-name}

## Purpose

Describe the system boundary, its responsibilities, and the problems it solves.

## Context

Identify upstream systems, downstream dependencies, and external actors.

## Components

List the major services, modules, or infrastructure elements and their responsibilities.

## Data Flow

Describe how requests, events, or records move across the system.

## Reliability Considerations

Document scaling, failover, resilience, and operational constraints.

## Security Considerations

Explain trust boundaries, authentication, authorization, and sensitive data handling.

## Open Questions

List unresolved architectural risks or decisions.
