# Integration Architecture: {integration-name}

## Purpose

Describe why the integration exists and which systems it connects.

## Participants

List the systems, services, teams, or external vendors involved.

## Interface Contracts

Document the APIs, events, file formats, or protocol contracts used.

## Sequence Of Interactions

Describe the end-to-end interaction flow and key decision points.

## Failure Modes

Document retry behavior, compensating actions, and known failure conditions.

## Security And Compliance

Explain data classification, access controls, and audit requirements.

## Monitoring

List logs, dashboards, alerts, and integration health indicators.
